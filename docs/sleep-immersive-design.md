# 睡眠模式「文字+音乐沉浸式体验」设计文档

## 一、方案概述

### 背景
当前睡眠模式已有白噪音和睡前文字两个独立功能，用户希望把这两个功能深度整合，形成沉浸式的睡眠体验。

### 设计理念
- 保持两个功能独立，但提供「一键联动」的沉浸式体验
- 支持两种使用模式：
  1. **纯模式**：只用文字 或 只用音乐
  2. **混合模式**：阅读时播放背景音乐

## 三、UI改动

### 1. 睡眠首页（沉浸式模式入口）
在现有3个模块卡片上方，添加「沉浸式模式」专区：
- 移动端：8个小卡片横向滚动
- 桌面端：4x2网格布局
- 每个卡片包含：图标 + 模式名称 + 一句简短描述
- 点击后跳转到对应页面（文字页+自动播放音乐）

### 2. 睡前文字页（背景音乐控制区）
右上角添加浮动的迷你控制区：
- 状态：🎵 背景音乐（当前播放的名称）或 🔇 无背景音乐
- 点击展开/收起迷你播放器
- 包含：播放/暂停、音量、声音切换、关闭

### 3. 音乐记忆（localStorage存储）
记住用户偏好：
- 上次播放的声音
- 音量设置
- 最后使用的沉浸式模式

## 四、交互流程

### 场景启动流程
1. 用户在首页点击「雨夜读信」卡片
2. 自动跳转到「睡前文字」页，默认到「信件」Tab
3. 自动开始播放雨声
4. 右上角控制区显示「🌧️ 雨声」

### 音乐切换流程
1. 在文字页点击右上角控制区展开
2. 选择其他声音（支持所有白噪音）
3. 页面背景色微动（呼应所选声音的色调）

## 七、边界与错误处理

1. **页面切换时音乐**：从文字页切换到其他页时，音乐继续播放（但进入白噪音页时自动接管）
2. **浏览器不支持AudioContext**：提示用户使用最新浏览器
3. **音乐播放失败**：安静降级，显示无法播放提示
4. **localStorage不可用**：不影响功能，只无法记住偏好
